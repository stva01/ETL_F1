fact_lap_time is the largest table (billions of rows) вЂ” partition by year and cluster by race_key.
Keep a separate fact_lap_summary mart for typical queries.
Cold storage for raw S3 files: S3 Glacier after 30 days; only Gold/Platinum tables stay in the hot

рџ“ђ STAR SCHEMA CONVENTIONS
Surrogate keys on all dims (BIGINT auto-increment). Never join on business keys in prod queries.
Conformed dimensions: dim_driver is shared across facts. One source of truth.
SCD Type 2 on dim_constructor (team name/nationality can change вЂ” see
Benettonв†’Renaultв†’Lotusв†’Renault).
Grain is explicit: Every fact table has a grain comment in DDL and in the dbt model YAML.
No NULLs in FK columns вЂ” use a "Unknown" surrogate key (в€’1) for late-arriving dims.
Immutable facts: Historical race results never change. Only standings facts may be overwritten.
рџ”„ INCREMENTAL LOAD PATTERN
After each race: S3 drop triggers Airflow DAG в†’ PySpark reads new CSVs в†’ MERGE into staging tables
(upsert on natural key).
dbt runs in --select tag:incremental mode вЂ” only rebuilds models downstream of changed source
tables.
Fact tables use dbt incremental materialisation with unique_key = result_id. New rows append;
corrected rows merge.
Marts rebuild fully after each race batch (cheap вЂ” they're aggregations of bounded-size data).
рџ¤– MCP / AI DESIGN CONSTRAINTS
Every mart table has a human-readable description in the semantic_catalog table.
The mart_ai_context wide table is a single-row-per-driver denormalised view вЂ” designed to fit in an
LLM context window for "tell me about Hamilton" type queries.
A nl_prompt_store table maps common question patterns to pre-built SQL, reducing AI hallucination on
schema details.
All column names are fully spelled out (no abbreviations) so the MCP layer can describe them without a
separate lookup.

вњ… DATA QUALITY GATES
Row count check: New race must have 16вЂ“24 result rows (modern grid size). Pipeline fails if outside
range.
Points validation: Sum of points in a race must match expected total for the era (e.g., top-10 points
system post-2010).
Referential integrity: Every result's driver_id and constructor_id must exist in dim tables before fact
load.
Duplicate guard: MERGE on (race_id, driver_id) prevents double-loading the same race.
Freshness alert: If staging tables haven't received new rows within 12 hours of race day, PagerDuty alert
fires.
